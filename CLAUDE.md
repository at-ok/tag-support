# UIデザイン思想

## マテリアルデザイン準拠のUI設計

本プロジェクトでは、Googleの**Material Design 3**の原則に基づいたUIデザインを採用しています。直感的で使いやすく、視覚的に美しいインターフェースを実現することを目標としています。

### 設計原則

#### 1. **Elevation（高さ）とシャドウ**
- マテリアルデザインの階層構造を表現するため、6段階のシャドウレベルを定義
- カード、ボタン、モーダルなどにCSS変数で管理されたシャドウを適用
- ホバー時にエレベーションを上げることで、インタラクティブ要素を強調

```css
--shadow-1: 0 1px 2px 0 rgba(0, 0, 0, 0.05);
--shadow-2: 0 1px 3px 0 rgba(0, 0, 0, 0.1), 0 1px 2px -1px rgba(0, 0, 0, 0.1);
--shadow-3: 0 4px 6px -1px rgba(0, 0, 0, 0.1), 0 2px 4px -2px rgba(0, 0, 0, 0.1);
```

#### 2. **カラーシステム**
- **Primary**: 青系（#2563eb）- アクションボタンや重要な要素
- **Success**: 緑系（#16a34a）- 成功状態やポジティブなアクション
- **Error**: 赤系（#dc2626）- エラーや警告、削除アクション
- **Warning**: オレンジ系（#ea580c）- 注意が必要なアクション
- **Surface**: 白・グレー系 - 背景とコンテンツ領域

各色には`:hover`と`:active`状態の変化を定義し、ユーザーフィードバックを強化

#### 3. **タイポグラフィ**
- Material Designのタイポグラフィスケールに準拠
- 見出し（h1-h6）に明確な階層構造を適用
- font-weightとletter-spacingで視認性を向上

#### 4. **8dpグリッドシステム**
- 余白とスペーシングは8pxの倍数を基準に設定
- 一貫性のあるレイアウトとリズムを実現

```css
--spacing-1: 0.5rem;  /* 8px */
--spacing-2: 1rem;    /* 16px */
--spacing-3: 1.5rem;  /* 24px */
--spacing-4: 2rem;    /* 32px */
```

#### 5. **モーションとアニメーション**
- **リップル効果**: ボタン押下時の視覚的フィードバック
- **トランジション**: 200-250msの滑らかな遷移
- **ease-out**: 自然な減速カーブでユーザー体験を向上
- **アクセシビリティ**: `prefers-reduced-motion`でモーション軽減に対応

#### 6. **タッチフレンドリー**
- すべてのインタラクティブ要素は**最小48px**のタッチターゲット（WCAG 2.1準拠）
- モバイルファーストのアプローチ
- セーフエリア対応（ノッチ付きデバイス）

#### 7. **アクセシビリティ**
- **High Contrast Mode**: 高コントラストモードで境界線を強調
- **Reduced Motion**: アニメーションを無効化するオプション
- **フォーカス状態**: キーボードナビゲーションのための明確なフォーカスリング
- **カラーコントラスト**: WCAG AAレベル以上のコントラスト比を維持

### コンポーネント設計

#### ボタン（Buttons）
- **Elevated Button**: 影付きの立体的なボタン（主要アクション用）
- **Outlined Button**: アウトライン型（副次的アクション用）
- **リップル効果**: クリック時の白いオーバーレイで視覚的フィードバック

#### カード（Cards）
- **card-mobile**: 基本カード（elevation-2）
- **card-elevated**: 強調カード（elevation-4）
- ホバー時にシャドウを増加させてインタラクション可能性を示唆

#### テキストフィールド（Text Fields）
- **アウトラインスタイル**: 2pxのボーダーで明確な境界
- **ホバー＆フォーカス**: 色の変化とシャドウで状態を表現
- **48px最小高**: タッチ操作に適した大きさ

#### チップ/バッジ（Chips/Badges）
- **丸みを帯びた形状**: rounded-fullで柔らかい印象
- **状態別カラー**: active（緑）、inactive（グレー）、captured（赤）、safe（青）

### レスポンシブデザイン

- **モバイルファースト**: 基本はモバイル向けに設計し、デスクトップに拡張
- **グラデーション背景**: 視覚的な深みを追加
- **フレキシブルグリッド**: grid/flexboxで柔軟なレイアウト

### 実装ガイドライン

#### 新規コンポーネント追加時
1. `card-mobile`または`card-elevated`をベースに使用
2. ボタンは`btn-primary`, `btn-success`, `btn-danger`, `btn-warning`, `btn-secondary`から選択
3. 入力フィールドは`input-touch`クラスを適用
4. 適切なelevationクラス（`elevation-1`～`elevation-6`）を使用
5. CSS変数（`--md-primary`など）でカラーを管理

#### スペーシング
- コンポーネント内の余白: `p-4`（16px）
- 要素間の間隔: `space-y-3`または`gap-3`（12px）
- セクション間: `space-y-4`（16px）

#### カラー適用
- プライマリアクション: `btn-primary`（青）
- 成功/開始: `btn-success`（緑）
- 危険/終了: `btn-danger`（赤）
- 警告/一時停止: `btn-warning`（オレンジ）
- 副次的: `btn-secondary`（アウトライン）

### 今後の拡張

- **ダークモード対応**: CSS変数を活用した簡易的なテーマ切り替え
- **カスタムアイコン**: SVGアイコンシステムの導入
- **より複雑なアニメーション**: フレーマーモーションなどのライブラリ検討
- **コンポーネントライブラリ化**: 再利用可能なコンポーネントの体系化

---

このデザインシステムは、**直感的な操作性**と**視覚的な美しさ**を両立し、ユーザーに快適な体験を提供することを目指しています。
